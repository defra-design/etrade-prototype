This API provides endpoints related to Export Health Certificates.

You can use the development api endpoint to test through the portal pages. To use the sandbox api endpoints you must use a registered application.

Read API specification

This API enables traders to apply for export health certificates, check the status of applications you’ve made, and check for validation errors on your applications. This is the first step towards enabling digital integration for moving agri-food goods.

An export health certificate (EHC) is an official document that confirms your export meets the health requirements of the destination country.

You must apply for an EHC via this API if you’re exporting or moving live animals or animal products from Great Britain (England, Scotland and Wales) to, or through:

  • The European Union
  • Non European Union countries
  • Northern Ireland (NI)

Further guidance on how to get an Export health certificate is available.

Currently Supported EHCs

This API is currently a alpha version and only supports the submission of a restricted list of EHC application forms. The supported list can be found here.

Coming Soon

We will soon be releasing more supported EHC’s via the API. Below is a list of EHC’s you can expect to see soon:

  • Great Britain to Northern Ireland
  • Great Britain to European Union

When to use this API

Use this API when you wish to apply for an export health certificate, which is required as part of the end to end process for exporting goods from Great Britain as detailed in step 4 in the high-level process below.

  1. Decide to export an animal or animal product
  2. Access guidance and understand EHC requirements for your consignment
  3. Contact your certifier/ OV who will be responsible for certifying your EHC before you
    4. submit
  4. your application Check status for the application
  5. Submit your application for an EHC*
  6. APHA review EHC application
  7. Physical EHC issued with wet signature
  8. Import notification is raised with destination authority
  9. The physical EHC will accompany the consignment as it leaves the UK

* This API enables system to system calls enabling integration with traders supply chain or logistics management application software, to facilitate the process of applying for an Export Health Certificate, reducing the burden of data entry duplication.

Usage scenarios

In this sandbox, this API currently provides the following endpoints enabling you too:

  • Submit a new Export Health Certificate application
  • Get validation errors for a submitted application
  • Get the status for a submitted Export Health Certificate application by request Id
  • Get metadata required for a submission for an Export Health Certificate
  • Additional endpoints are provided for development and testing purposes to enable you to:
  • Retrieve sample payloads for supported Export Health Certificates

Find an OV or Inspector for your Export

An official veterinarian is someone who has been authorised by APHA to perform statutory duties (work that is required by law). Use this guide to find a professional to certify your export health certificate.

As part of your application, you will need to include a specified OV or local authority inspector, ensuring that you:

  • Name them in your application
  • Check that they can inspect your consignment and sign your EHC in the days before your export

Defra will be providing, in a future iteration of this service, a reference data API to publish an approved list of OV/local authority inspectors to enable you to attach the relevant details to your application.

User Authentication

The endpoint in this API is user-restricted and application-restricted. Your users, before submitting transactions via the API, must have registered to use the Export Health Certificate online service, https://www.access.service.gov.uk/login/signin/creds.
